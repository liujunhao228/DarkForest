# 黑暗森林 AI Agent 测试用例集

## 1. 测试目标

确保黑暗森林 AI Agent 的功能完整性、稳定性和用户体验质量，覆盖以下关键方面：

- **核心功能测试**：AI 模型调用、响应处理、数据输入输出验证
- **边界条件测试**：异常输入处理、网络错误恢复、资源限制场景
- **用户界面交互测试**：各组件响应性、操作流程完整性
- **性能和可靠性测试**：响应时间、稳定性、容错能力

## 2. 测试环境

### 2.1 硬件环境
- CPU: 至少 4 核心
- 内存: 至少 8GB
- 网络: 稳定的本地网络

### 2.2 软件环境
- Python 3.11+
- nanobot (本地 LLM 服务)
- 游戏服务器 (Bun/Socket.IO)
- 依赖包: `uv sync --all-extras`

### 2.3 环境配置
```bash
# 复制环境配置
cp .env.example .env

# 启动 nanobot
nanobot serve

# 启动游戏服务器
bun run src/server/gameServer.ts
```

## 6. 常见问题和解决方案

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| LLM 连接失败 | nanobot 未启动 | 运行 `nanobot serve` |
| 游戏服务器连接失败 | 游戏服务器未启动 | 运行 `bun run src/server/gameServer.ts` |
| 匹配失败 | 玩家不足 | 启动多个 AI Agent |
| JSON 解析失败 | LLM 返回格式错误 | 检查 Prompt 格式和 LLM 配置 |
| 操作验证失败 | AI 决策不符合规则 | 调整 Prompt 或 LLM 参数 |
