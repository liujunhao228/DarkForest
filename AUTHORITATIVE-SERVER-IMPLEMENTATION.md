## 关键特性

### ✅ 防作弊
- 所有游戏逻辑在服务器端
- 客户端只能发送操作请求
- 服务器验证每个操作

### ✅ 状态同步
- 全量同步（初始/版本差距大）
- 增量同步（正常游戏时）
- 自动版本追踪

### ✅ 断线重连
- 客户端重新连接后请求全量同步
- 服务器保留房间状态

### ✅ 并发控制
- 操作排队处理
- 防止竞态条件

## 数据流

### 正常游戏流程
```
1. 玩家 A 出牌
   └─> 客户端发送: socket.emit('game:action', { action: 'playCard', payload: {...} })
   
2. 服务器接收
   └─> EventHandlers 路由到 RoomManager
   └─> RoomManager 获取房间和引擎
   └─> AuthoritativeGameEngine 调用 GameValidator 验证
   └─> 验证通过，执行操作
   └─> 计算状态变化
   └─> StateSyncManager 广播给所有玩家
   
3. 客户端更新
   └─> 接收 game:fullSync 或 game:deltaSync
   └─> 应用状态
   └─> 更新 UI
```

## 部署说明

### 本地开发
```bash
# 启动 WebSocket 服务器
bun run src/server/gameServer.ts

# 启动 Next.js 开发服务器
bun run dev
```

### 生产部署
```bash
# 构建
bun run build

# 启动（需要同时运行两个服务）
./.zscripts/production-start.sh
```

### Docker 部署
```dockerfile
FROM oven/bun:1
WORKDIR /app
COPY . .
RUN bun install && bun run build
EXPOSE 3000 3003
CMD ["bun", "run", "start:all"]
```
